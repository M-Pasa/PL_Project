## 1. Valid Programs

### 1.1 `valid_01_full.fl` — complete bulk-phase program

```fitlang
athlete(bodyweight: 78 kg, goal: bulk) {

    workout "Push Day" {
        exercise "Bench Press"    { sets: 4, reps: 6, weight: 80 kg }
        exercise "Overhead Press" { sets: 3, reps: 8, weight: 50 kg }
        progress weight 2.5 kg/week
    }

    meal "Breakfast" { protein: 40 g, carbs: 80 g, fat: 15 g }

    target protein >= 1.8 g per kg of bodyweight
    goal   gain 0.5 kg/week

    when goal: bulk {
        target protein >= 2.0 g per kg of bodyweight
    } else {
        target protein >= 1.6 g per kg of bodyweight
    }

    plan week with workouts: ["Push Day"], meals: ["Breakfast"]
}
```

**`--run` output:**

```
FitLang Plan
  Athlete: bodyweight = 78 kg, goal = bulk
  Horizon: 1 week
  Goal rate: gain at 0.5 kg/week
  Week 1 (bodyweight = 78 kg):
    Workouts:
      Push Day:
        Bench Press: 4 × 6 @ 80 kg
        Overhead Press: 3 × 8 @ 50 kg
    Meals:
      Breakfast: protein 40 g carbs 80 g fat 15 g
    Macros (total): protein 40 g carbs 80 g fat 15 g
    Targets:
      protein >= 1.8 g per kg of bodyweight (140.4 g): planned 40 g → OFF (-100.4 g)
      protein >= 2 g per kg of bodyweight (156 g): planned 40 g → OFF (-116 g)
```

**Discussion.** This program exercises the full §4.3 skeleton: workout, meal, all four rule-decl alternatives, and the mandatory schedule-stmt. The `when goal: bulk` selection (§4.4.4) is resolved at compile time — the `>= 2.0 g` branch is kept and the `else` branch is dead-code-eliminated — so the interpreter sees exactly one target per the athlete's declared goal. The per-bodyweight threshold `2.0 × 78 kg = 156 g` demonstrates §4.4.3 target-scaling; both targets appear in the output because the static rule is added first and the when-selected rule second.

### 1.2 `valid_02_workout_only.fl` — routine template with parameter substitution

```fitlang
athlete(bodyweight: 90 kg, goal: cut) {

    routine "Base Press"(load: kg, accessory_load: kg) {
        exercise "Bench Press"    { sets: 4, reps: 6, weight: load }
        exercise "Overhead Press" { sets: 3, reps: 8, weight: accessory_load }
    }

    workout "Pull Day" {
        exercise "Deadlift"    { sets: 3, reps: 5, weight: 140 kg }
        exercise "Barbell Row" { sets: 4, reps: 8, weight:  70 kg }
    }

    workout "Push Day" {
        use "Base Press"(82 kg, 22 kg)
        exercise "Triceps Pushdown" { sets: 3, reps: 12, weight: 30 kg }
    }

    goal lose 0.75 kg/week
    plan week with workouts: ["Pull Day", "Push Day"]
}
```

**`--run` output:**

```
FitLang Plan
  Athlete: bodyweight = 90 kg, goal = cut
  Horizon: 1 week
  Goal rate: lose at 0.75 kg/week
  Week 1 (bodyweight = 90 kg):
    Workouts:
      Pull Day:
        Deadlift: 3 × 5 @ 140 kg
        Barbell Row: 4 × 8 @ 70 kg
      Push Day:
        Bench Press: 4 × 6 @ 82 kg
        Overhead Press: 3 × 8 @ 22 kg
        Triceps Pushdown: 3 × 12 @ 30 kg
```

**Discussion.** `"Base Press"` is declared with two `kg`-typed parameters and invoked via `use "Base Press"(82 kg, 22 kg)`, which §4.4.3 (PLAN, routine-call expansion) evaluates by binding `load = 82 kg` and `accessory_load = 22 kg` in an extended environment and re-evaluating the routine body — the result appears as `Bench Press @ 82 kg` and `Overhead Press @ 22 kg`. The `<intake-section>` is absent so `<body-sections>` selects the workout-only alternative, producing a `Plan` with no macro totals or target rows; this confirms that the intake section is optional per §4.3.2.

### 1.3 `valid_04_multi_week.fl` — four-week progression

```fitlang
athlete(bodyweight: 78 kg, goal: bulk) {

    workout "Push Day" {
        exercise "Bench Press"    { sets: 4, reps: 6, weight: 80 kg }
        exercise "Overhead Press" { sets: 3, reps: 8, weight: 50 kg }
        progress weight 2.5 kg/week
    }

    meal "Breakfast" { protein: 40 g, carbs: 80 g, fat: 15 g }

    target protein >= 1.8 g per kg of bodyweight
    goal   gain 0.5 kg/week

    plan 4 week with workouts: ["Push Day"], meals: ["Breakfast"]
}
```

**`--run` output (weeks 2–3 omitted; they follow the same pattern):**

```
FitLang Plan
  Athlete: bodyweight = 78 kg, goal = bulk
  Horizon: 4 week
  Goal rate: gain at 0.5 kg/week
  Week 1 (bodyweight = 78 kg):
    Workouts:
      Push Day:
        Bench Press: 4 × 6 @ 80 kg
        Overhead Press: 3 × 8 @ 50 kg
    Macros (total): protein 40 g carbs 80 g fat 15 g
    Targets:
      protein >= 1.8 g per kg of bodyweight (140.4 g): planned 40 g → OFF (-100.4 g)
  Week 4 (bodyweight = 79.5 kg):
    Workouts:
      Push Day:
        Bench Press: 4 × 6 @ 87.5 kg
        Overhead Press: 3 × 8 @ 57.5 kg
    Macros (total): protein 40 g carbs 80 g fat 15 g
    Targets:
      protein >= 1.8 g per kg of bodyweight (143.1 g): planned 40 g → OFF (-103.1 g)
```

**Discussion.** `plan 4 week` invokes the §4.4.3 PLAN rule with N = 4, projecting bodyweight as `bw₀ + 0.5(k−1) kg` and exercise weights as `base + 2.5(k−1) kg`, so Bench Press advances `80 → 82.5 → 85 → 87.5 kg` across weeks 1–4. Per-bodyweight targets re-evaluate each week against the projected bodyweight (`1.8 × 78 = 140.4 g` at week 1; `1.8 × 79.5 = 143.1 g` at week 4), confirming that target thresholds are week-dependent functions of the bodyweight projection rather than static constants.

## 2. Malformed Programs

Each fails at parse time; exit code 1.

### 2.1 Empty athlete body

```fitlang
athlete(bodyweight: 78 kg, goal: bulk) {
    plan week with workouts: ["Push Day"]
}
```
```
Parse error: Expected 'workout' or 'meal' block inside athlete block [line 3, col 5]
```
`<body-sections>` is one-or-more; `plan` is outside its FIRST set so no derivation covers an empty body (§4.3.2).

### 2.2 Missing `<schedule-stmt>`

```fitlang
athlete(bodyweight: 78 kg, goal: bulk) {
    workout "Push Day" {
        exercise "Bench Press" { sets: 4, reps: 6, weight: 80 kg }
    }
}
```
```
Parse error: Expected PLAN but got RBRACE ('}') [line 5, col 1]
```
`<schedule-stmt>` is mandatory and last in `<athlete-block>` (§4.3.3); its absence leaves `}` where `plan` is expected.

### 2.3 Invalid goal mode

```fitlang
athlete(bodyweight: 78 kg, goal: shred) {
    workout "Push Day" {
        exercise "Bench Press" { sets: 4, reps: 6, weight: 80 kg }
    }
    plan week with workouts: ["Push Day"]
}
```
```
Parse error: Expected goal mode: bulk, cut, or maintain [line 2, col 34]
```
`<goal-mode>` is a three-value enumeration; `shred` emits `IDENT` and fails all three alternatives (§4.2.3).

## 3. Type-Error Program

### `invalid_typeerror.fl` — five violations, one checker pass

```fitlang
athlete(bodyweight: 78 kg, goal: bulk) {

    let bad_sum   = 80 kg + 200 kcal         # cross-family + (§4.5.4)
    let bad_ident = unknownVar               # unbound identifier (§4.6.3)

    workout "Push Day" {
        exercise "Bench" { sets: 4, reps: 6, weight: 80 kcal }  # wrong family (§4.5.5)
        progress weight 2.5 kcal/week                            # wrong rate (§4.5.5)
    }

    goal gain 0.25 g/kg                     # denominator not Time (§4.5.5)

    plan week with workouts: ["Push Day"]
}
```

**Checker output (stderr, exit code 1):**

```
Type errors in 'samples\invalid_typeerror.fl':
  - let bad_sum: operator '+' is not defined on Quantity(MASS) and Quantity(ENERGY) (§4.5.4)
  - let bad_ident: unbound identifier 'unknownVar'
  - workout "Push Day" exercise "Bench" weight: must be Quantity(MASS), got Quantity(ENERGY)
  - workout "Push Day" progress rate: must be Rate(MASS, TIME), got Rate(ENERGY,TIME)
  - goal gain rate: must be Rate(MASS, TIME), got Rate(MASS,MASS)
```

All five diagnostics are emitted in one AST pass (let-section first, then workouts, rules, schedule); the interpreter is never invoked. Each message names the violated §4.5/§4.6 rule.

## 4. Runtime-Error Programs

Both programs pass parsing and type-checking; the interpreter rejects them during `--run`.

### `runtime_negative_weight.fl`

```fitlang
athlete(bodyweight: 80 kg, goal: bulk) {
    let benchBase = 80 kg
    let bigCut    = 90 kg
    workout "Push Day" {
        exercise "Bench Press" { sets: 4, reps: 6, weight: benchBase - bigCut }
    }
    plan week with workouts: ["Push Day"]
}
```
```
Runtime error: exercise "Bench Press" weight evaluated to -10 kg at week 1 (must be ≥ 0)
```
`benchBase - bigCut` is well-typed (`Quantity(MASS)`) but evaluates to a negative value; sign tracking requires dependent types outside Sebesta Ch. 6, so the interpreter checks every computed Mass slot at run time (§4.4.3, R1).

### `runtime_negative_bodyweight.fl`

```fitlang
athlete(bodyweight: 30 kg, goal: cut) {
    workout "Push Day" {
        exercise "Bench Press" { sets: 4, reps: 6, weight: 40 kg }
    }
    goal lose 2 kg/week
    plan 20 week with workouts: ["Push Day"]
}
```
```
Runtime error: bodyweight projected to 0 kg at week 16 (must be > 0);
  goal lose at 2 kg/week over plan 20 week with starting bodyweight 30 kg
```
`30 kg − 2 kg/week × 15 week = 0 kg` is not statically detectable; the §4.4.3 PLAN loop checks each projected bodyweight before the week's evaluation and aborts at the first non-positive result (§4.4.3, R2).
